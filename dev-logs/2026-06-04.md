## 关键发现

- `ELECTRON_RUN_AS_NODE=1` 由 VS Code / Claude Code 终端注入，导致 Electron 二进制运行在纯 Node.js 模式
- 该模式下 `process.type` 为 `undefined`（正常应为 `browser`）
- `require('electron')` 返回路径字符串而非 API 对象
- 解决方案：在 `electron` 命令前添加 `ELECTRON_RUN_AS_NODE=`（设为空字符串）
- 国内镜像（npmmirror）下载的 Electron zip 与官方版本一致，结构无缺失
- Electron 28 的 `electron.asar` 内嵌在 `Electron Framework` 二进制中（143MB），不存在独立 .asar 文件
