# 温暖纪念册 UI 与静默开机启动设计

## 目标

将主界面和桌面小组件统一改为“温暖纪念册”视觉风格，并取消用户可见的开机启动开关。应用默认注册为登录时启动；由 macOS 登录触发时静默启动，仅显示桌面小组件，不弹出主界面。

## 范围

### 开机启动

- 删除主界面的“开机启动：开/关”按钮及对应前端状态和事件处理。
- 应用正常启动后自动确保 `openAtLogin` 为开启状态。
- 用户手动打开应用时，显示主界面和桌面小组件。
- macOS 登录触发应用启动时，不显示主界面，仅创建并显示桌面小组件。
- 用户可继续通过小组件标题或右键菜单打开主界面。
- 如果系统拒绝注册登录项，应用继续正常运行，不因设置失败而退出。

### 主界面

- 采用温暖米白背景、陶土红重点色和低对比暖灰文字。
- 标题、操作区和内容区建立清晰层级，减少当前按钮堆叠感。
- 顶部突出“下一个重要日子”的倒计时信息。
- 纪念日卡片采用柔和纸张质感，强化名称、日期和倒计时，次要信息降低视觉权重。
- 搜索、新增、面板开关、编辑、删除、置顶、导入导出等现有能力保持可用。
- 保留清晰的键盘焦点、足够的文字对比度和点击区域。

### 桌面小组件

- 与主界面使用同一套米白、陶土红和暖灰配色。
- 使用暖色半透明面板和轻量阴影，保持桌面悬浮感。
- 倒计时作为每条纪念日的主要视觉信息，名称和备注保持易读。
- 保留拖动、滚动、隐藏单条、恢复全部和打开主界面的交互。
- 不改变现有小组件尺寸、位置持久化、显示状态持久化和刷新机制。

## 启动流程

1. Electron `ready` 后确保应用被注册为登录项。
2. 读取当前启动是否由系统登录项触发。
3. 始终创建桌面小组件。
4. 仅在非登录项启动时创建并显示主界面。
5. 用户从小组件请求打开主界面时，按需创建主窗口并显示。

## 不在本次范围

- 不改动纪念日数据结构、存储路径和备份格式。
- 不改动公历、农历、周年循环和通知计算规则。
- 不引入图片素材、在线字体或新的运行时依赖。
- 不改造成 WidgetKit 原生小组件。

## 错误处理

- 注册开机启动失败时记录错误，但不阻断应用、小组件或主界面。
- 主窗口按需创建失败时沿用现有错误处理方式，不影响已有数据。
- UI 改造不删除或迁移用户数据。

## 验证标准

- 主界面不再出现开机启动按钮。
- 应用启动后登录项设置为开启。
- 模拟登录项启动时主界面不显示，小组件正常显示。
- 手动启动时主界面和小组件均正常显示。
- 从小组件可以打开主界面。
- 新增、编辑、删除、置顶、搜索、筛选、拖动排序、导入导出和小组件显隐功能通过回归验证。
- 自动化测试、生产构建和应用打包通过。
