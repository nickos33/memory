# Memory 可靠性与周年提醒修复设计

## 目标

在不丢失现有用户数据的前提下，修复筛选拖拽导致的数据删除、损坏文件被覆盖、异常备份导入、周年提醒失效和小组件状态不同步，并把项目收敛为单一可维护入口。

## 数据兼容

- 旧记录没有 `recurrence` 时按 `yearly` 处理，符合纪念日应用的主要用途。
- 新记录支持 `yearly` 与 `once`；默认 `yearly`。
- 公历继续保留 `date`，每年重复时使用其中的月、日计算下一次发生日期。
- 农历记录新增 `lunarMonth`、`lunarDay`、`lunarLeap`。旧农历记录从原 `date` 对应的农历日期补齐这些字段。
- 2 月 29 日在非闰年按 2 月 28 日处理。

## 模块边界

- `src/shared/memorial-date.mjs`：日期解析、农历换算、下一次发生日期、倒计时和展示文本。主进程与两个 React 界面共用。
- `src/shared/memorial-data.mjs`：记录规范化、字段校验、稳定重排和导入合并，不访问文件系统。
- `store.mjs`：只负责读取、备份、原子写入和调用数据模块。
- `notifications.mjs`：只负责定时调度与通知展示，日期判断交给共享日期模块。

## 数据安全

- 文件不存在时返回空列表；文件存在但无法解析时抛出可识别错误，禁止后续覆盖。
- 每次写入先写同目录临时文件，再原子替换；覆盖前保留 `memorials.json.bak`。
- 导入必须逐条校验对象、ID、名称、日期、事由、标签及农历字段；任一无效记录都会拒绝整次导入并返回可读错误。
- 重排只更新传入记录的 `order`，从不删除未参与排序的记录。

## 用户界面

- 新增/编辑表单提供“每年重复／仅一次”选项。
- 农历模式直接选择农历月、日和是否闰月，不再把公历日期输入框伪装成农历输入。
- 主窗口与小组件使用下一次发生日期排序和显示倒计时。
- 搜索覆盖人名、事由和备注；无结果与无数据使用不同文案。
- 所有异步失败显示给用户，按钮在操作期间禁用。

## 小组件与窗口

- 面板显示/隐藏都写回 `widget-config.json`。
- 小组件订阅主进程刷新事件并清理监听器。
- “恢复全部”同时清空 React 状态和 localStorage。
- Dock 激活时显示并聚焦已有主窗口。

## 安全与可维护性

- 只保留 `main.mjs`、`store.mjs`、`notifications.mjs`、`preload.cjs` 这一套运行入口，不再打包漂移的备用实现。
- HTML 增加本地资源 CSP；BrowserWindow 禁止新窗口和外部导航；IPC 校验发送方和参数。
- 图标按钮提供可访问名称，表单标签与控件关联，弹窗支持 Escape。

## 验证
