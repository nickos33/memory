# Memory — 纪念日提醒

一款 macOS 桌面应用，帮助你记录、浏览和回顾重要纪念日，并在纪念日临近时发送系统通知。拥有一个常驻桌面的深色毛玻璃悬浮小组件，让你随时查看即将到来的纪念日，无需打开主窗口。

## 特性

- **纪念日管理** — 新增、编辑、删除纪念日，支持农历日期
- **置顶与排序** — 重要纪念日置顶显示，支持拖拽排序
- **桌面悬浮小组件** — 深色毛玻璃悬浮面板，常驻桌面。支持拖拽移动、右键上下文菜单、单条纪念日显隐切换
- **临近提醒** — 每天上午 9:00 自动检查，纪念日前 7 天、3 天、1 天和当天发送系统通知
- **搜索筛选** — 按人名、事由实时搜索，支持标签筛选
- **数据备份** — 支持导出/导入 JSON 备份文件，数据完全由你掌控
- **开元自启** — 注册为 macOS 登录项，开机自动启动
- **完全离线** — 所有数据存储在本地，无需网络连接

## 系统要求

- macOS 10.15 或更高版本
- Apple Silicon (arm64)

## 快速开始

### 环境要求

- Node.js >= 18
- npm >= 9

### 安装与运行

```bash
# 克隆项目
git clone <repo-url> && cd 纪念日

# 安装依赖（注意：需要先清除 ELECTRON_RUN_AS_NODE 环境变量）
ELECTRON_RUN_AS_NODE= npm install

# 构建 + 打包 + 启动
npm start
```

### 构建安装包

```bash
# 构建 .dmg 安装包
npm run build

# 仅打包（不生成 .dmg）
npm run pack
```

## 操作手册

### 主窗口

![主窗口示意图]

主窗口是纪念日的管理中心。打开 app 后首先看到的是主窗口，也可以从小组件右键菜单中打开。

#### 查看纪念日

所有纪念日以卡片列表形式纵向排列。置顶卡片排在顶部。临近的纪念日（今天/明天）会高亮显示。点击"搜索"按钮展开搜索栏，输入关键词实时过滤；还可以按标签分类筛选。

#### 新增纪念日

点击右上角"+ 新增"按钮，在弹出的表单中填写：

| 字段 | 说明 | 必填 |
|------|------|------|
| 人名 | 纪念对象的名字 | 是 |
| 日期 | 公历或农历日期 | 是 |
| 事由 | 选择预设类型或自定义输入 | 是 |
| 备注 | 任意补充信息 | 否 |
| 标签 | 自定义标签，方便筛选 | 否 |

日期切换为"农历"后，系统每年自动计算对应的公历日期。

#### 编辑与删除

鼠标悬停在卡片上会显示操作菜单（置顶/编辑/删除）。编辑会打开表单弹窗，删除前会弹出确认对话框——删除操作不可恢复。

#### 拖拽排序

使用拖拽库（@dnd-kit），直接拖拽卡片即可调整同组内（置顶/非置顶）的排列顺序，排序会自动持久化。

#### 面板按钮

主窗口顶部有一个"📌 面板"按钮，点击可切换桌面悬浮小组件的显示/隐藏。

#### 搜索按钮

蓝色胶囊按钮，点击展开搜索栏，支持关键词 + 标签双重筛选。

### 桌面悬浮小组件

小组件是 Memory 的核心交互亮点。它是一块深色毛玻璃风格的小面板，始终悬浮在桌面上方，让你无需切换窗口就能看到所有纪念日。

#### 视觉效果

- 深色毛玻璃背景（`rgba(42, 58, 80, 0.78)` + `backdrop-filter: blur(18px)`）
- 标题"Memory"采用花体（Zapfino / Snell Roundhand），居中显示
- 每条纪念日为独立圆角卡片，鼠标悬停时背景变亮
- 过去的纪念日以半透明显示，未来的清晰显示；今天/明天高亮
- 小组件关闭按钮和红绿灯已移除，只保留右键菜单

#### 位置与大小

- 小组件位置固定为 260×190 px（不可调整大小）
- 拖拽移动小组件到任意位置，位置自动保存到 `widget-config.json`
- 默认位置：屏幕左上角 (20, 20)

#### 交互方式

| 操作 | 效果 |
|------|------|
| 拖拽面板空白区域 | 移动小组件位置 |
| 点击标题"Memory" | 打开主窗口 |
| 点击任意纪念日卡片 | 打开主窗口 |
| 右键纪念日卡片 | 切换该纪念日的显示/隐藏 |
| 右键面板空白区域 | 打开上下文菜单 |
| 点击底部"已隐藏 N 个" | 一键恢复所有隐藏的纪念日 |

#### 右键菜单

右键小组件面板弹出以下选项：

- **打开主窗口** — 显示主窗口并聚焦
- **恢复全部隐藏** — 将被隐藏的纪念日全部恢复显示
- **隐藏面板** — 隐藏小组件（可通过主窗口"📌 面板"按钮重新打开）
- **退出应用** — 完全退出 Memory

#### 隐藏与恢复

- **隐藏单条：** 右键点击任意纪念日卡片，该卡片从小组件中隐藏
- **恢复全部：** 如果隐藏了任意数量纪念日，底部会出现"已隐藏 N 个 · 点击恢复全部"的提示，点击即可恢复
- **右键恢复：** 右键面板 →"恢复全部隐藏"
- **主窗口同步检测：** 小组件每 30 秒自动从主窗口同步数据；在主窗口中新增/编辑/删除纪念日后小组件即时刷新

### 通知系统

- 每天上午 9:00 自动检查所有纪念日
- 匹配范围：当天、提前 1 天、提前 3 天、提前 7 天
- 点击通知直接跳转到主窗口

### 数据备份

- **导出：** 主窗口底部有"导出"按钮，将全部数据保存为 JSON 文件
- **导入：** 主窗口底部有"导入"按钮，读取 JSON 文件合并到现有数据（自动过滤重复项）
- 建议定期导出备份，文件默认命名为 `memorials-backup-YYYY-MM-DD.json`

## 边界条件

### 显示边界

| 场景 | 行为 |
|------|------|
| 无纪念日记录 | 小组件显示"暂无纪念日"，主窗口显示空状态引导 |
| 所有纪念日被隐藏 | 小组件仅显示"已隐藏 N 个 · 点击恢复全部" |
| 所有纪念日为过去 | 全部以 50% 不透明度显示，仍可正常交互 |
| 搜索无匹配 | 卡片列表区域显示空结果提示 |
| 窗口最小尺寸 | 主窗口 600×400 px，小组件不可调整大小 |

### 数据边界

| 场景 | 行为 |
|------|------|
| 数据文件不存在（首次启动） | 自动初始化为空列表 |
| 数据文件损坏（JSON 解析失败） | 降级为空列表，不报错 |
| 导入的文件非 JSON 格式 | `JSON.parse` 抛出异常，需手动处理 |
| 导入的 JSON 不合数据模型 | 仍然导入（不做校验），可能导致界面显示异常 |
| 重复 ID | 导入时自动过滤已存在的 ID |
| 小组件配置文件不存在 | 使用默认位置 (20, 20) |

### 小组件边界

| 场景 | 行为 |
|------|------|
| 小组件被隐藏后重启 app | 保持隐藏状态，直到用户通过主窗口按钮打开 |
| 小组件窗口被操作系统关闭 | `widgetWindow` 引用置为 null，下次 toggle 时重新创建 |
| 同时显示多个小组件 | **不支持**，每个时刻只有一个小组件实例 |
| 多显示器 | `setVisibleOnAllWorkspaces(true)`，小组件在所有空间可见 |
| 农历纪念日跨年 | `lunar-typescript` 自动计算每年公历日期 |

### 通知边界

| 场景 | 行为 |
|------|------|
| macOS 通知权限未授予 | `Notification.isSupported()` 返回 false，静默跳过 |
| 同一日多个纪念日 | 每条独立触发通知 |
| app 未运行（进程不存在） | 通知不会触发（需要 app 在后台运行） |

### 退出与后台

| 场景 | 行为 |
|------|------|
| 关闭主窗口（点红 × 或 Cmd+W） | 主窗口隐藏，小组件保持显示，app 不退出 |
| Cmd+Q 或 Dock 右键退出 | `forceQuitting = true`，主窗口和小组件全部销毁，app 退出 |
| 非 macOS 平台 | `window-all-closed` 事件直接退出 app |

## 开发

```bash
# 安装依赖
ELECTRON_RUN_AS_NODE= npm install

# 构建 + 打包 + 启动
npm start

# Webpack watch 模式 + Electron 启动
npm run dev

# 构建 .dmg 安装包
npm run build

# 仅打包（不生成 .dmg）
npm run pack
```

> 如果 `npm install` 后 `node_modules/electron/dist/Electron.app` 缺失，可以手动从国内镜像下载并解压：
> ```bash
> # 手动下载 Electron 二进制
> ELECTRON_MIRROR="https://npmmirror.com/mirrors/electron/" \
>   node -e "const {downloadArtifact}=require('@electron/get');downloadArtifact({version:'28.2.0',artifactName:'electron',platform:'darwin',arch:'arm64',mirrorOptions:{mirror:'https://npmmirror.com/mirrors/electron/'}}).then(console.log)"
> # 解压到 node_modules/electron/dist/
> unzip -q <下载的zip路径> -d node_modules/electron/dist/
> ```

### 已知问题

- 若 shell 环境持久设置了 `ELECTRON_RUN_AS_NODE=1`，会阻止 `electron` npm 包的二进制下载。`package.json` 的 scripts 已显式清除该变量
- `node_modules/electron/index.js` 与 Electron 内置模块冲突，开发时需通过打包后运行，不能用 `npx electron .` 直接启动
